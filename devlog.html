🐛 双 Bug 修复
修复页面卡在"正在加载模型"

✏️ 手动标注编辑 Bug 修复
修复手动画框后无法拖动、点选、调整
  • 根因 1：mousedown 只在 iconImg && sel 时才设 dragFaceIdx，手动框被忽略
  • 根因 2：mousemove 的 if (drawMode) return 在拖拽处理之前短路
  • 修复：mousedown 只要 hitIdx ≥ 0（或 hitGear ≥ 0）就设 dragFaceIdx，两种模式通用
  • 修复：mousemove 把 拖拽更新提前到 drawMode 分支之前执行
  • 拖拽仅在 iconImg && face.sel 时更新位置，其余只记录 isDragging
  • mouseup 的 dragFaceIdx 分支在 draw/select 两种模式下均可触发点击 / 切换 sel / 开关面板
  • 光标离开 draw 模式后正确恢复 crosshair
11
🔧 单独偏移不生效 Bug 修复
修复拖动单独偏移滑块，贴纸纹丝不动
  • 根因 1：mpfor（单独偏移滑块）缺少 input 事件监听器，拖动无任何反应
  • 根因 2：renderStickerCanvas（最终导出渲染）只用全局 offsetPct，忽略了 d.offsetOv
  • 根因 3：resetPerFace 只清除 scaleOv，未清除 offsetOv，重置后偏移残留
  • 修复：补充 mpfor 的 input 监听，实时写入 d.offsetOv 并调用 drawModal()
  • 修复：renderStickerCanvas 改为 d.offsetOv != null ? d.offsetOv/100 : globalOffPct
  • 修复：resetPerFace 同时 delete scaleOv 和 offsetOv
