🎭 贴纸换脸 · 开发全记录

1
🚀 第一版：基础功能搭建
新功能纯前端 · 单 HTML 文件
  • 上传自定义贴纸图标（PNG/GIF/WebP）
  • 批量上传照片（拖拽 / 文件选择）
  • 使用 tinyFaceDetector 检测人脸
  • 自动将图标覆盖到所有检测到的人脸
  • 图标大小 / 偏移滑块调节
  • 逐张下载 / 全部下载
2
🎯 升级检测 + 人脸选择
问题无法选择替换哪些脸；检测精度差
  • 换用 ssd_mobilenetv1，精度更高
  • 置信度阈值 0.25，减少漏检
  • 新增弹窗选脸模式：点击人脸框切换绿/灰
  • 绿色 = 会贴贴纸   灰色 = 跳过
  • 默认全选，用户主动取消不需要的脸
3
🖼️ 修复点击 + 放大 UI
问题点击人脸框无反应；照片太小
  • face-api.js Box 对象 getter 可能失效
    → 改为存储纯数字 {x, y, w, h}
  • 点击命中区加 pad 容差，不必精准点边框
  • 卡片按原图比例展示，不强制方形裁剪
  • 弹窗全屏展示（92vh），图片放大清晰
  • 鼠标悬停人脸框有 hover 高亮，光标变手型
4
👁️ 弹窗内实时预览
体验应用后大小不合适，调整再应用还是旧尺寸
  • 滑块移入弹窗，拖动即时重绘 canvas
  • 有图标时显示贴纸预览；无图标时显示选框
  • 选中人脸悬停 → 暗色遮罩 + "点击取消"文字
  • 未选人脸悬停 → "点击添加"提示
  • 确认时将 stickerScale / stickerOffset 保存到图片对象
5
✅ 两步确认流程
体验确认后返回页面应该显示调整后效果
  • 弹窗按钮改为 「确认选择」（仅保存，不生成）
  • 确认时立即用当前参数渲染 previewCanvas
  • 卡片缩略图立即显示预览结果（橙色边框提示待应用）
  • 卡片底部出现 「✨ 应用贴纸」 按钮 + 「重新选脸」
  • 「应用贴纸」直接复用 previewCanvas，不重复计算
6
🔍 侧脸检测 + 手动标注
问题侧脸、远处人脸漏检；无法手动添加
  • 置信度阈值降至 0.2
  • 镜像翻转二次检测：图片水平翻转后再跑一遍，坐标映射回原图
  • 合并两次结果 + NMS 非极大值抑制去重（IoU > 0.35）
  • 新增 「✏️ 手动标注」模式：拖拽绘制橙色框
  • 右键任意人脸框 → 删除（清除误检 / 公司内部人员）
7
🐛 双 Bug 修复
修复页面卡在"正在加载模型"
  • 语法错误：绘制矩形代码误插到顶层作用域（多余 }），导致整个 script 块解析失败、所有函数未执行
  • 模型加载挂起：CDN 请求 stall（不报错也不完成）→ 加 20s 超时 + Promise.race
  • 超时后显示 「🔄 重试」按钮，一键重新加载
